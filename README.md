# 🎯 PlacementAI — AI Placement Preparation System

A complete, local-first AI-powered placement preparation platform. Runs fully on CPU.

---

## 🚀 Features

| Feature | Description |
|---|---|
| 🎤 **AI Interview Simulator** | Voice-based interview with real-time face & eye tracking proctoring |
| 👁️ **Face/Eye Proctoring** | OpenCV detects if you look away — 5+ warnings terminate the session |
| 🗣️ **Speech-to-Text** | Browser Web Speech API for recording answers |
| 🤖 **AI Answer Evaluation** | Gemini AI scores your answers on content, communication & depth |
| 🧠 **AI Quiz Generator** | Fresh MCQs generated by AI every session across 8 CS topics |
| 💻 **Coding Lab** | 6 curated problems (Easy→Hard) with AI hints, review & test execution |
| 📊 **Performance Dashboard** | Track study time, quiz scores, problems solved, interview sessions |
| 🏢 **Company Prep Guides** | AI-generated prep guide for any company (Google, TCS, Infosys, etc.) |
| 📄 **Resume Analyzer** | AI scores your resume, finds missing keywords, gives action items |
| 📝 **Mock Test** | Simulated full placement exam flow |
| 📚 **Study Material** | 8 CS topics (DBMS, SQL, DSA, OOP, CN, OS, Aptitude, Coding) |

---

## 📋 Requirements

- Python 3.8+
- Webcam + Microphone (for interview mode)
- Internet connection for Gemini AI features, or local Ollama for offline/local AI
- Optional Google Gemini API key (free at [aistudio.google.com](https://aistudio.google.com))
- Optional Ollama install for local LLM fallback

**Works on:** Windows · macOS · Linux · CPU only (no GPU needed)

## ⚡ Quick Start

### 1. Clone / Extract the project
```bash
cd placement_system
```

### 2. Set up AI backend
```bash
cp .env.example .env
# Option A: edit .env and add your GEMINI_API_KEY
# Option B: run a local LLM with Ollama
ollama pull qwen2:0.5b
ollama serve
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Run
```bash
python run.py
# OR
python app.py
```

### 5. Open browser
```
http://localhost:5000
```

## ☁️ Deploy Free on Render

This project includes `render.yaml` and `Procfile`, so Render can run it as a free Python web service.

### Option A: Use `render.yaml`
1. Push this project to GitHub.
2. Go to [https://dashboard.render.com](https://dashboard.render.com).
3. Click **New +** → **Blueprint**.
4. Connect your GitHub repository.
5. Render will read `render.yaml` and create the free web service.
6. Add `GEMINI_API_KEY` when Render asks for environment variables.

### Option B: Create a Web Service manually

Use these settings:

| Setting | Value |
|---|---|
| Runtime | Python 3 |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120` |
| Instance Type | Free |

Add these environment variables in Render:

```env
SECRET_KEY=your-random-secret
GEMINI_API_KEY=your-gemini-api-key
```

Render Free services can sleep after inactivity, so the first visit after a pause can take around a minute to load.

## 🔑 Getting a Free Gemini API Key

1. Go to [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
2. Sign in with your Google account
3. Click "Create API Key"
4. Copy the key to your `.env` file:
   ```
   GEMINI_API_KEY=AIza...
   ```

The free tier is sufficient for all features in this system.

## 🧠 Local LLM Fallback

SparkLearn falls back to Ollama when `GEMINI_API_KEY` is missing or Gemini cannot generate a response.

1. Install Ollama from [https://ollama.com](https://ollama.com)
2. Pull the default lightweight model:
   ```bash
   ollama pull qwen2:0.5b
   ```
3. Start Ollama before running SparkLearn:
   ```bash
   ollama serve
   ```
4. Optional `.env` settings:
   ```
   OLLAMA_HOST=http://localhost:11434
   OLLAMA_MODEL=qwen2:0.5b
   ```

## ⚠️ Troubleshooting

**Camera not working?**
- Allow camera/microphone permissions in browser
- Use Chrome or Edge for best Web Speech API support

**AI features not working?**
- If using Gemini, check your `GEMINI_API_KEY` in `.env` and ensure internet access
- If using local fallback, run `ollama serve` and pull the configured `OLLAMA_MODEL`

**OpenCV error?**
- Run: `pip install opencv-python-headless`

**Port already in use?**
- Change `PORT=5001` in `.env`
